# The Book of Heresies

Things considered doctrinally unsound.

## Architecture Heresies

- **"We'll fix it later."** — Later never comes. Fix it when you discover it.
- **"Adding AI because AI."** — Technology follows requirement, not the other way around.
- **"Mistaking confidence for evidence."** — Confidence is not data.
- **"Starting with Kubernetes."** — Solve the problem you have, not the one you theorize.

## Design Heresies

- **"Everything should be customizable."** — Customization is complexity. Build strong defaults.
- **"Maximize engagement metrics."** — Optimize for actual outcomes, let metrics follow.
- **"Move fast and break things."** — Break things that matter and you pay a cost.

## The Master Heresy

**"Thou shalt mistake a coherent diagram for reality."**

A beautiful model is not evidence that it matches reality.
